	-> Törlés esetén az ügyfelet mentse el az alkalmazás egy audit (archív) táblába

2) Jog (szerep) bevezetése
	-> adatbázisba migrálni admin jogot
	-> adatbázisba migrálni tanácsadói jogot

3) Index módosítása 
	-> csak tanácsadói joggal rendelkező felhasználó érhesse el

4) Admin felületek létrehozása
	-> minden ügyfelet lát
	-> minden ügyfelet tud törölni és szerkeszteni
	-> tud törölni véglegesen (biztonsági mentés nélkül)
	-> minden felhasználót lát
	-> tudja módosítani a felhasználók jogosultságát
